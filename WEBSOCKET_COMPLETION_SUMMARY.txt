## 💡 Key Concepts Explained

### Thread Safety
- **ConcurrentHashMap**: Allows multiple threads to safely access room data
- **synchronized blocks**: Ensures multi-step operations are atomic
- **volatile fields**: Guarantees all threads see latest value
- **Platform.runLater()**: Safely updates UI from background thread
- **CountDownLatch**: Synchronizes connection before returning

### Message Flow
1. Client clicks bid
2. Sends JSON to server (non-blocking)
3. Server processes bid safely (synchronized method)
4. Server broadcasts result to all clients
5. Client receives on WebSocket thread
6. Switches to JavaFX thread with Platform.runLater()
7. Updates UI labels, shows alerts

### Why This Matters
Without thread safety:
- Multiple bids might corrupt data
- UI might update from wrong thread → exceptions
- Race conditions → unpredictable behavior
- App crashes

With thread safety (what you have now):
- All operations are safe
- Data is consistent
- UI always updates correctly
- No crashes or exceptions
